## Pipeline and data integrity findings

### Fixed

1. The collectors were sequential and could block the pre-market run. They now
   use a bounded `asyncio.to_thread` pool with global request-start spacing.
2. External requests had one-shot failure behavior. Transient 408/425/429/5xx,
   timeouts, and connection failures now use bounded exponential backoff and
   honor `Retry-After`.
3. Tiingo's `--api-key` was parsed but never passed to the request function.
4. Tiingo failure had no fallback. Output now records either `tiingo`,
   `yfinance`, or `unavailable` as the actual source.
5. Earnings `--all-tickers` expected an obsolete object layout rather than the
   current `tickers` array.
6. SEC used an Atom query that omitted Form 4. It now consumes SEC's official
   submissions JSON and constructs direct filing archive URLs.
7. R2 accepted malformed and non-standard JSON. All private artifacts now reject
   NaN/Infinity; core AI artifacts additionally require Pydantic envelopes.
8. File writes could be observed half-written. Collector output now uses fsync +
   atomic replace.
9. The npm tree reported one moderate and two high transitive advisories. Next
   is pinned to 15.5.21 with safe PostCSS 8.5.10 and Sharp 0.35.0 overrides;
   final `npm audit --omit=dev` reports zero vulnerabilities.

### Architectural boundary

The live five-model council executor and learning loop are not in this
repository; `README.md` identifies `arshad1416/hermes-scripts` as their home.
`pi-scripts/council_weighting.py` therefore provides PR-ready, provider-neutral
logic rather than pretending it is wired here. Integration in `hermes-scripts`
must map provider IDs to the stable mandates, pass scored regime-tagged outcomes,
and consume the normalized weights before aggregation.

The algorithm uses Bayesian smoothing, 45-day recency decay, matching-regime
evidence, bounded weights, and explicit affinity for high VIX, pin-heavy expiry,
macro catalysts, and low-VIX momentum. No model can dominate on a tiny sample.
